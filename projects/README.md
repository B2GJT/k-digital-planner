# Projects 입력 가이드

## 먼저 확인하세요

입력은 각 프로젝트 폴더의 `inputs.txt`와 `.xlsx` 파일에 합니다. 루트의 `run-project.bat` 메뉴에 표시되려면 프로젝트 폴더 안에 `inputs.txt`와 `.xlsx` 파일이 모두 있어야 합니다.

필수로 입력해야 하는 항목은 `과정명`, `카테고리`입니다. `AOP 링크`는 선택입니다. AOP 링크가 없으면 `inputs.txt`의 강사/커리큘럼/기획 메모를 중심으로 분석합니다.

## 주의사항

- 항목명 뒤에는 반드시 콜론(`:`)을 붙여주세요. 예: `과정명: AICE Associate 자격증 대비반 with Python`
- 항목 순서는 바뀌어도 됩니다. 코드는 항목명으로 인식합니다.
- 항목명은 가급적 이 README의 표기를 그대로 사용해주세요.
- 여러 줄 입력은 항목명 다음 줄부터 이어서 작성하면 됩니다. 다음 `항목명:`이 나오기 전까지 같은 항목의 내용으로 인식합니다.
- 빈 항목은 삭제하지 않아도 됩니다. 예: `AOP 링크:`처럼 비워둘 수 있습니다.
- `카테고리`는 아래 “사용 가능한 카테고리” 중 하나를 권장합니다.
- `.xlsx` 파일은 프로젝트 폴더 바로 아래에 넣어주세요. 파일명이 `cr.xlsx`이면 우선 사용하고, 없으면 첫 번째 `.xlsx` 파일을 사용합니다.
- 임시 엑셀 파일(`~$...xlsx`)은 무시됩니다.
- AOP 페이지가 로그인, 차단, 자바스크립트 렌더링을 요구하면 텍스트 추출이 실패할 수 있습니다. 이 경우 `inputs.txt`에 강사, 주요 내용, 기획 메모를 직접 보완해주세요.
- 권장 길이는 항목당 2,000자 이내입니다. 길어도 실행은 되지만 Codex 호출 시간이 늘고 결과가 산만해질 수 있습니다.
- 실행 중 각 단계가 끝나면 결과 파일을 확인하고 다음 단계 진행 여부를 묻습니다. 계속하려면 Enter 또는 `Y`, 중단하려면 `n`을 입력하세요.

## 프로젝트 폴더 구조

```text
projects/
└─ my-project/
   ├─ inputs.txt
   ├─ cr.xlsx              # 또는 다른 .xlsx 파일
   ├─ run.bat
   ├─ extracted/           # 자동 생성
   └─ results/             # 자동 생성
```

실행 중 생성되는 오류 로그와 Codex 전달 프롬프트는 프로젝트 폴더 안이 아니라 루트의 `logs/<프로젝트폴더명>/`에 저장됩니다. 예: `projects/gemini`의 로그 저장 위치는 `logs/gemini/`입니다.

단계별 Codex 지침은 루트의 `prompts/` 폴더에서 관리합니다. 개별 프로젝트 폴더 안에 프롬프트를 둘 필요는 없습니다.

## inputs.txt 항목

- `과정명`: 필수. 생성할 과정 또는 강의명입니다.
- `카테고리`: 필수. 아래 카테고리 중 하나를 권장합니다.
- `AOP 링크`: 선택. 입력하면 해당 페이지를 추출해 AOP 분석에 사용합니다.
- `강사`: 선택. AOP에서 강사 정보가 추출되지 않을 때 보완 자료로 사용합니다.
- `러닝타임`: 선택. CR에서 확인되지 않을 때 보완 자료로 사용합니다.
- `커리큘럼`: 선택. CR에서 확인되지 않을 때 보완 자료로 사용합니다.
- `파이널 프로젝트`: 선택. 최종 결과물 또는 수강생 산출물 방향입니다.
- `기획 메모`: 선택. 상세페이지/기획안에서 강조할 방향입니다.
- `강조하고 싶은 Edge`: 선택. 차별화 포인트 생성 시 우선 반영합니다.
- `제외하고 싶은 표현`: 선택. 기획안에서 피해야 할 진부한 표현입니다.

## 사용 가능한 카테고리

- AI/머신러닝
- 데이터 분석
- 업무 자동화
- 디지털 마케팅
- 클라우드/인프라
- 웹 개발
- 사무 자동화(Excel/PPT/Word)
- 영상 편집/콘텐츠 제작
- 사이버보안
- UX/UI 디자인
- 기타

## inputs.txt 예시

```text
과정명: AICE Associate 자격증 대비반 with Python
카테고리: AI/머신러닝
AOP 링크:

강사:
홍길동 / 데이터 분석 및 AI 교육 경력 7년 / Python, pandas, 머신러닝 실무 프로젝트 다수

러닝타임:
총 40시간

커리큘럼:
Chapter 1. Python 데이터 분석 기초
- Python 문법, 데이터 구조, 파일 입출력
Chapter 2. pandas 기반 데이터 전처리
- 결측치, 이상치, 그룹 분석, 시각화
Chapter 3. AICE 유형별 문제 풀이
- 분류, 회귀, 평가 지표, 모델 개선

파이널 프로젝트:
AICE Associate 실전형 문제를 기반으로 데이터 분석 리포트와 예측 모델을 완성

기획 메모:
자격증 취득만 강조하지 말고, 비전공 실무자가 실제 업무에서 데이터 분석을 시작할 수 있다는 점을 함께 강조

강조하고 싶은 Edge:
실전 문제 풀이, 비전공자 친화 설명, 즉시 재사용 가능한 분석 템플릿

제외하고 싶은 표현:
체계적인 커리큘럼, 실무 중심, 전문 강사진
```
